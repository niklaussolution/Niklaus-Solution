1. # FEATURE OVERVIEW

The Seminar Management feature provides a complete system for:

- Promoting seminars on the public website
- Collecting seminar registrations from students and employees
- Storing registration data securely in Firestore
- Managing registrations through an admin dashboard
- Exporting registration data for reporting

This is a fully integrated feature with professional UI/UX across all pages.

================================================================================ 3. FIRESTORE DATABASE STRUCTURE
================================================================================

Collection: seminarRegistrations

Document Fields:

- id (Auto-generated by Firestore)
- name (String) - Full name of registrant
- email (String) - Email address
- phone (String) - Mobile number
- role (String) - "Student" or "Employee"
- organization (String) - College/Company name
- city (String) - City/Location
- registeredAt (Timestamp) - Firestore server timestamp
- status (String) - "registered" (future: can be extended)

Example Document:
{
id: "abc123xyz",
name: "John Doe",
email: "john@example.com",
phone: "9876543210",
role: "Student",
organization: "ABC College",
city: "Mumbai",
registeredAt: Timestamp(2026-04-14T10:30:00Z),
status: "registered"
}

Total Documents Expected: Varies with registrations

================================================================================ 7. USER WORKFLOWS
================================================================================

7.1 VISITOR REGISTRATION WORKFLOW

1. Visitor lands on home page
2. Scrolls to see SeminarSection (promotes the seminar)
3. Clicks "Register Now" button in SeminarSection
4. Redirected to /seminar page
5. Views seminar details and information
6. Sees registration form on the right
7. Fills out all required fields:
   - Full Name
   - Email
   - Phone Number
   - Role (Student/Employee)
   - Organization
   - City
8. Clicks "Complete Registration"
9. Form validates all required fields
10. Shows loading spinner while submitting
11. Data saved to Firestore (seminarRegistrations collection)
12. Confirmation email sent via EmailJS
13. Success message displayed with congratulations
14. Form clears automatically
15. Email sent with confirmation details

7.2 ADMIN MANAGEMENT WORKFLOW

1. Admin logs into admin dashboard
2. Navigates to sidebar → "Seminar Registrations"
3. Sees dashboard with:
   - Statistics (Total, Students, Employees)
   - Search functionality
   - Role filtering
   - Sortable table with all registrations
4. Can perform actions:
   - Search by name, email, or organization
   - Filter by role
   - View all registration details
   - Select registrations (checkboxes)
   - Export to CSV file
   - Delete individual registrations
   - Refresh data
5. CSV Export includes:
   - Name, Email, Phone, Role, Organization, City, Registered Date
   - Filename format: seminar-registrations-YYYY-MM-DD.csv

Environment Variables Required:

- FIREBASE_AUTH_DOMAIN
- FIREBASE_DATABASE_URL
- FIREBASE_STORAGE_BUCKET
- FIREBASE_MESSAGING_SENDER_ID
- FIREBASE_APP_ID

Firebase Setup:

1. Create 'seminarRegistrations' collection (auto-created on first write)
2. Set Firestore security rules (see below)
3. Enable Firestore in Firebase console

Firestore Security Rules (Recommended):

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Allow public writes to seminarRegistrations
    match /seminarRegistrations/{document=**} {
      allow write: if true;
      allow read: if request.auth != null && request.auth.token.admin;
    }
  }
}
```

================================================================================ 17. TROUBLESHOOTING
================================================================================

Issue: Form submission fails
Solution:

- Check Firebase configuration
- Verify Firestore is enabled
- Check browser console for errors
- Verify EmailJS keys are correct

Issue: Registrations not appearing in admin
Solution:

- Check Firestore security rules allow reads
- Verify admin has correct role
- Check collection name: "seminarRegistrations"
- Try manual refresh button

Issue: Email not being sent
Solution:

- EmailJS is non-critical (registration still saves)
- Check EmailJS credentials
- Verify email template exists
- Check service ID and template ID

Issue: Mobile form not displaying
Solution:

- Check responsive breakpoints
- Clear browser cache
- Test in mobile simulator
- Verify Tailwind CSS is loaded
