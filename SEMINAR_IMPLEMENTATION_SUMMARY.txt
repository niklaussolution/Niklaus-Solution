================================================================================

Document Structure:
{

  email: String (email address),

✅ Delete (admin can remove registrations)
✅ Export (CSV download with all data)

Query Performance:
- Ordered by registeredAt descending (newest first)

SECURITY IMPLEMENTATION
================================================================================

Public Registration:
✅ No authentication required (open to all)
✅ Form validation on client-side
✅ Firestore enforces server-side rules
✅ HTTPS only (Vercel)

Admin Access:
✅ Firebase authentication required
✅ Role-based access control (RBAC)
✅ Protected route component checks roles
✅ Only super_admin and editor can access
✅ Firestore rules enforce permissions

Data Protection:
✅ Firestore encryption at rest
✅ User data never exposed in URLs
✅ Email/phones not logged in plain text
✅ Timestamps server-side (prevents manipulation)

Recommended Firestore Rules:
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Public write for registrations
    match /seminarRegistrations/{document=**} {
      allow create: if true;
      // Admin read/delete
      allow read, delete: if request.auth != null && 
        get(/databases/$(database)/documents/admins/$(request.auth.uid)).data.role 
        in ['super_admin', 'editor'];
    }
  }
}
```

SUPPORT & TROUBLESHOOTING
================================================================================

Common Issues & Solutions:

Issue: "Cannot find module SeminarPage"
Solution: Verify import path in main.tsx is correct
        Check file exists at src/app/pages/SeminarPage.tsx
        Restart dev server

Issue: Form submission fails silently
Solution: Check browser console (F12) for errors
        Verify Firebase credentials in config/firebase.ts
        Check Firestore is enabled in console
        Verify security rules allow writes

Issue: Registrations not appearing in admin
Solution: Check you're logged in as admin
        Verify admin role is "super_admin" or "editor"
        Check Firestore collection name matches
        Try manual refresh button
        Check browser console for errors

Issue: EmailJS not sending
Solution: This is non-critical - registrations still save
        Check service/template IDs in code
        Verify EmailJS account is active
        Check email template syntax
        Test in EmailJS console first

Issue: CSV export not working
Solution: Check registrations exist in table
        Try filtering to ensure data loads first
        Check browser console for JavaScript errors
        Try exporting fewer registrations
        Try different browser

Issue: Mobile layout broken
Solution: Clear browser cache (Ctrl+Shift+Delete)
        Check viewport meta tag in index.html
        Verify Tailwind CSS loads correctly
        Test in mobile device simulator
        Check for CSS conflicts
